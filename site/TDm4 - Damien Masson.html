    Threads et synchronisation avec smaphores

    Objectifs

    • Mettre en oeuvre des programmes multithreads
    • Mettre en oeuvre des mcanismes de synchronisation de programmes multithreads reposant sur les smaphores

    Remarques prliminaires

    • La commande make permet de compiler un programme dont les sources sont rparties sur plusieurs fichiers. Pour une description simple et suffisante pour nos sances de make et des fichiers makefile, on peut lire ce tutoriel.
    • Il existe de nombreux outils trs utiles au dveloppement regroups dans le projet GNU. Vous pouvez lire ce document d'introduction aux outils de dveloppement GNU.
    • Dans la suite de l'nonc on appellera bibliothque un ensemble compos d'un fichier .c et d'un fichier .h pouvant contenir des dfinitions de type, des fonctions et des procdures mais pas de fonction main().

    #include <unistd.h>
#include <stdlib.h>
#include <stdio.h>
#include <string.h>

#include "gestionFichiers.h"

char * litLigne(int fd)
{
    int i;
    int nbr;
    char buf[TAILLEBUF];
    char * s;

    for(nbr = 0 ; nbr < TAILLEBUF ; nbr++){
        int rc = read(fd, buf+nbr,1);
        if( rc == 0  ){ /*  fin de fichier */
            return NULL;
        }
        if(rc== -1){/* erreur */
            perror("erreur de lecture dans litLigne");
            return NULL;
        }

        if(buf[nbr]=='\n')break;
    } 

    s=(char*)malloc(nbr+1);
    if(s==NULL){
        perror("alocation issue in litLigne");
        return NULL;
    }

    for(i=0;i<nbr;i++)
        s[i]=buf[i];
    s[i] = '\0';
    return s;
}

/* retourne -1 en cas d'echec, taille de la chaine sinon */
int ecritLigne(char* chaine,int fd)
{

    int size_s = strlen(chaine);
    int nbw=0;
    int tmp;

    while(nbw!=size_s){
        tmp = write(fd,chaine+nbw,size_s-nbw);
        if(tmp==-1){
            perror("write issue in ecritLigne");
            return -1;
        }
        nbw+=tmp;
    }
    tmp = write(fd,"\n",1);/* on ne met pas le \n dans chaine car ne marcherait pas avec une chaine statique */
    if(tmp==-1){
        perror("write issue in ecritLigne");
        return -1;
    }
    return nbw;

}

    A. Implmentation brute force

    Implantez le schma ci-dessus.

    La mmoire partage sera un buffer de char dclar comme variable globale par : 

    char buffer[MAX];

    Si vous tes en avance vous pouvez proposer une solution o le buffer est allou dynamiquement dans le fils d'excution principal et transmis aux threads concurrents via les paramtres de leur routine d'excution.

    B. On rflchit

    Quel est le problme de la solution A ? Proposez (sans l'implmenter) un schma de solution au problme de A avec un mutex.

    C. On rflchit encore puis on programme

    Quel est le problme avec votre solution prcdante ? Proposez une solution avec deux smaphores.

    Attention, votre programme doit fonctionner quelque soit la taille du buffer d'change partag entre les thread. Si le buffer d'change est plus petit que la taille d'une ligne, cela veut dire que le thread 2 doit pouvoir lire les caractres un par un dans le buffer partag, sans pour autant les crire (puisqu'il crit avec ecritLigne() une ligne entire). Il faudra certainement penser utiliser un buffer local supplmentaire dans le thread 2... De plus le thread 2 doit pouvoir se terminer lorsque le thread 1 n'crit plus. Pour cela le thread 1 pourra terminer par un ajout du caractre spcial EOF dans le buffer partag.

    Compilation

    Pour compiler un programme qui se sert des threads, il faut ajouter l'option -lpthread la commande d'dition des liens.
